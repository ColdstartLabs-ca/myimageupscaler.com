# SEO Ongoing Maintenance - Phase 6

This directory contains templates and procedures for ongoing SEO maintenance and monitoring.

## Purpose

Phase 6 establishes processes for continuous SEO improvement after implementation phases are complete. This ensures SEO health is monitored and improved monthly rather than treated as a one-time project.

## Documents

### 1. Monthly GSC Report Template & Example

**Files:**
- `MONTHLY_GSC_REPORT_TEMPLATE.md` - Template for monthly reports
- `MONTHLY_GSC_REPORT_2026-02.md` - First completed example (baseline)

**What to Track Monthly:**
- Position changes for top queries
- New queries appearing
- Indexing progress (0/1,471 indexed at baseline)
- Click/impression trends
- Technical issues detected
- Device and geographic breakdown
- Cannibalization and duplicate issues

**Usage:**
1. Copy template each month: `MONTHLY_GSC_REPORT_YYYY-MM.md`
2. Export data from Google Search Console (28-day period)
3. Fill in metrics and analysis
4. Document actions completed and next month's priorities

### 2. Backlink Acquisition Tracking

**File:** `BACKLINK_ACQUISITION_TRACKING.md`

**What to Track:**
- Monthly new referring domains (target: 10+ per month)
- DR progress (baseline: DR 0)
- Outreach status for directories, guest posts, communities
- Link quality assessment
- Competitor backlink analysis

**Usage:**
- Weekly: Update outreach tracker
- Monthly: Review new domains acquired, check DR
- Quarterly: Run competitor backlink analysis

**Related Assets:**
- Directory Submission Checklist: `../saas-directory-submission/CHECKLIST.md`

### 3. AI Citation Monitoring

**File:** `AI_CITATION_MONITORING.md`

**What to Track Monthly:**
- ChatGPT citations
- Perplexity citations
- Google AI Overview mentions
- Claude AI citations

**Baseline:** 0 citations across all platforms

**Monthly Checklist:**
- Test queries on each AI platform
- Record mentions and positions
- Verify robots.txt allows AI bots
- Check /llms.txt accessibility
- Update schema markup as needed

### 4. Content Freshness Procedures

**File:** `CONTENT_FRESHNESS_PROCEDURES.md`

**What to Document:**
- When to update blog post dates
- Publication staggering (target: 2-3 posts/month)
- Content update prioritization
- Last modified date procedures

**Procedures Covered:**
- Date update workflow (when to change date vs lastModified)
- Publication calendar management
- Content refresh priorities (traffic/stale matrix)
- Redating existing content

## Monthly Workflow

### Week 1: Data Collection
- [ ] Export GSC data (previous 28 days)
- [ ] Check backlinks (Ahrefs or manual)
- [ ] Test AI platform queries
- [ ] Review content freshness metrics

### Week 2: Analysis
- [ ] Complete monthly GSC report
- [ ] Update backlink tracker
- [ ] Document AI citation findings
- [ ] Identify content needing updates

### Week 3: Action
- [ ] Execute backlink outreach
- [ ] Update high-priority content
- [ ] Fix technical issues identified
- [ ] Schedule next month's publications

### Week 4: Review & Plan
- [ ] Review month's SEO performance
- [ ] Update content freshness log
- [ ] Plan next month's priorities
- [ ] Prepare maintenance summary
